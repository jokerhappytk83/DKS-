## 화면 목록

### 1. 홈 화면 (Home)
- **내용**: 환영 메시지, 내 고충 현황 요약 카드 (접수중/처리중/완료 건수), 빠른 접수 버튼, 최근 고충 목록 미리보기
- **기능**: 고충 접수 화면으로 이동, 목록 화면으로 이동

### 2. 고충 접수 화면 (Submit)
- **내용**: 카테고리 선택 (근무환경/인사/급여/대인관계/기타), 제목 입력, 상세 내용 입력, 익명 여부 선택, 긴급도 선택, 첨부 이미지 (선택)
- **기능**: 폼 유효성 검사, 접수 완료 후 목록으로 이동

### 3. 고충 목록 화면 (List)
- **내용**: 내가 접수한 고충 목록 (카드형), 상태 필터 탭 (전체/접수중/처리중/완료), 각 항목에 카테고리 배지, 날짜, 상태 표시
- **기능**: 상태별 필터링, 상세 화면으로 이동

### 4. 고충 상세 화면 (Detail)
- **내용**: 접수 내용 전체, 처리 상태 타임라인, 담당자 코멘트, 첨부 이미지
- **기능**: 상태 확인, 취소 요청

### 5. 관리자 화면 (Admin)
- **내용**: 전체 고충 목록, 상태별 통계 차트, 미처리 건수 강조
- **기능**: 상태 변경 (접수→검토→처리중→완료/반려), 코멘트 작성

### 6. 설정 화면 (Settings)
- **내용**: 직원 정보 (이름, 부서, 사번), 알림 설정, 다크모드 토글
- **기능**: 프로필 편집, 테마 전환

## 핵심 사용자 플로우

### 고충 접수 플로우
홈 화면 → "고충 접수" 버튼 탭 → 접수 폼 작성 (카테고리, 제목, 내용) → 익명 여부 선택 → 제출 → 완료 확인 모달 → 목록 화면

### 처리 현황 확인 플로우
홈 화면 → "내 고충 목록" 탭 → 목록 화면 → 특정 항목 탭 → 상세 화면 (타임라인 확인)

### 관리자 처리 플로우
관리자 탭 → 미처리 목록 확인 → 항목 선택 → 상태 변경 + 코멘트 작성 → 저장

## 탭 바 구성
1. 홈 (house.fill)
2. 접수 (plus.circle.fill)
3. 내 목록 (list.bullet)
4. 관리자 (shield.fill)
5. 설정 (gearshape.fill)

## 레이아웃 원칙
- 모든 화면은 ScreenContainer 사용
- 카드 컴포넌트: rounded-2xl, shadow, border
- 상태 배지: 색상 코딩된 pill 형태
- 입력 폼: 충분한 패딩, 명확한 레이블
- 버튼: 주요 액션은 full-width, 보조 액션은 outline
